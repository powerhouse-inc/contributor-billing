# methodology

## Principles
- Prose-as-title: every insight is a proposition that reads naturally when linked
- Wiki links: connections as graph edges with semantic context
- Maps: attention management hubs for navigating knowledge areas
- Capture fast, distill slow — separate the two activities

## My Process

### Distill
Read source material (PRs, coding sessions, documentation) through the product lens. Extract patterns, decisions, and ideas worth keeping. Each insight gets a title that works as prose when linked.

### Connect
After distilling creates new insights, find relationships with existing knowledge. Update maps to include new material. Check whether older insights need updating in light of new discoveries.

### Revisit
Periodically return to older insights and ask: "If I wrote this today, what would be different?" The codebase evolves — insights must evolve with it.

### Validate
Check that summaries predict content, schema is complete, and wiki links resolve. Fix issues immediately.
